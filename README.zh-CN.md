# ClipQuery

[English](./README.md)

这是一个基于 Chromium 的选中文本问答插件。用户在网页中选中一段文本后，可以直接发起提问、解释、翻译、总结，并在浮动面板、网页内侧栏或浏览器原生侧栏中继续多轮对话。

## 项目亮点

- 选中文本后出现快捷操作栏：`提问`、`解释`、`翻译`、`总结`
- 支持浮窗、网页内侧栏、浏览器原生 `sidePanel`
- 支持多轮对话与流式输出
- 支持 Markdown 渲染与 KaTeX 数学公式渲染
- 支持收藏问答并导出为 Markdown 文件
- 支持多个 Provider 配置，并可按动作分别路由模型
- 支持三种回答策略：`严格网页模式`、`平衡模式`、`自由问答模式`
- 支持实验性的 Codex OAuth 登录

## 主要功能

- `提问`
  打开对话面板，并围绕当前选中文本继续追问。
- `解释`
  使用配置给“解释”动作的模型解释当前内容。
- `翻译`
  使用配置给“翻译”动作的模型输出译文。
- `总结`
  使用配置给“总结”动作的模型提炼要点。
- `固定`
  将面板与入口按钮固定在当前页面。
- `收藏`
  将一轮问题与回答导出为本地 Markdown 文件。

## Provider 支持

当前插件支持多个 Provider Profile。

- `OpenAI 兼容接口`
  适用于兼容 `chat/completions` 的接口，例如 OpenAI 兼容网关、Qwen 兼容接口等。
- `Codex OAuth（实验）`
  先完成一次 Codex 登录，再把指定动作路由到 Codex Profile。

每个 Provider 可以单独配置：

- 名称
- 类型
- API 地址
- 模型名称
- 温度
- 系统提示词
- API Key

## 回答策略

- `严格网页模式`
  更忠于当前网页选中文本，不主动补充缺失知识。
- `平衡模式`
  优先参考当前网页内容，但在用户明显是在问一个独立问题时，允许适度补充常识。
- `自由问答模式`
  更接近通用助手，网页选中文本只是辅助上下文。

## 安装方式

1. 打开 Chrome 或 Edge。
2. 进入扩展程序页面。
3. 开启“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择你克隆或下载后的项目目录。

## 配置方式

打开插件设置页后，建议完成以下配置：

- 新增一个或多个 Provider
- 为 `提问`、`解释`、`翻译`、`总结` 分别选择默认 Provider
- 选择默认回答策略

一个常见的搭配方式是：

- `提问` -> 更强的通用模型
- `解释` -> 推理能力更强的模型
- `总结` -> 更快更便宜的模型
- `翻译` -> 双语能力更好的模型

## 使用方式

1. 在网页中选中一段文本。
2. 点击快捷操作栏中的某个动作。
3. 在浮动面板、网页侧栏或浏览器侧栏中继续对话。
4. 按 `Ctrl+Enter` 可以快速发送。
5. 点击回答旁边的 `收藏` 可以保存为 Markdown 文件。

如果输入框为空，插件会尽量把当前选中文本本身视为问题。

## 数学公式渲染

插件已经支持常见数学公式输出格式。

- 块级公式：`\[ ... \]`、`$$ ... $$`
- 行内公式：`\(...\)`、`$...$`

公式渲染由 KaTeX 在插件内部完成。

## 项目结构

- `manifest.json`
  Manifest V3 插件配置
- `background.js`
  模型路由、Provider 解析、流式请求、Codex OAuth、收藏导出
- `content.js`
  选区监听、快捷栏、浮窗、网页内侧栏、聊天渲染
- `content.css`
  网页内 UI 样式
- `sidepanel.html`、`sidepanel.js`、`sidepanel.css`
  浏览器原生侧栏相关文件
- `options.html`、`options.js`、`options.css`
  设置页、Provider 管理、动作路由、回答模式配置

## 注意事项

- API Key 目前保存在浏览器扩展本地存储中。
- Codex OAuth 目前是实验性实现，基于公开可观察到的 Codex 客户端行为整理而来，并不是官方公开的浏览器扩展标准接入方案。
- OpenAI 兼容 Provider 使用 `chat/completions` 形态请求。
- Codex OAuth Provider 使用 Responses 风格请求。
- 如果准备正式发布，建议改为通过你自己的后端代理请求模型，而不是让插件直接持有凭据。

## 当前状态

项目主流程已经可用，但真实体验仍然会受到这些因素影响：

- 浏览器版本
- Provider 兼容性
- Codex OAuth 后端行为
- 模型输出格式差异
